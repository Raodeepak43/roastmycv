**Service-based company resume** — TCS, Infosys, Wipro, Cognizant, Accenture, HCL — hire tens of thousands of Indian freshers per cycle. Your CV hits an ATS, then a recruiter who has **10 seconds** and **300 more PDFs** in queue.

This is not the resume you send to a Bangalore product startup. Format and keywords differ.

## Service vs Product — Resume Difference

| Factor | Service (TCS, Infosys…) | Product startup |
|--------|-------------------------|-----------------|
| Length | 1 page fresher | 1 page OK, projects heavy |
| Design | Plain ATS | GitHub links matter more |
| CGPA | Often eligibility gate | More flexible |
| Keywords | Java, SQL, testing, support | Stack-specific depth |
| Photo | No | Rarely |
| Projects | 2 solid with stack | Live deployed apps win |

## Service Company Resume Template

```
NAME + Contact
─────────────────
SUMMARY (2 lines — role + CGPA/project hook)

EDUCATION
B.Tech | College | CGPA | Year | No active backlogs

TECHNICAL SKILLS
Languages | Databases | Tools | (match JD exactly)

PROJECTS
• [Name] — [Stack] — [Your role] — [Metric]
• [Name] — [Stack] — [Your role] — [Metric]

INTERNSHIPS / TRAINING
• [Company] — [Duration] — [2 bullets]

CERTIFICATIONS / ACHIEVEMENTS
• NPTEL / contest rank / hackathon
```

## Company-Specific Notes

### TCS
- NQT eligibility — CGPA, backlogs, gap rules
- [TCS NQT resume format](/blog/tcs-nqt-resume-format-fresher)

### Infosys
- Similar mass-hire ATS — keyword-heavy skills section
- Infosys-specific certifications help if relevant

### Wipro / Cognizant / Accenture
- Same one-page ATS rules
- Tailor summary line per role (developer vs tester vs support)

### HCL / Tech Mahindra
- Often same campus pipeline — reuse core format, swap keywords

## Keywords Service ATS Looks For

Pull from **exact job posting**:

- Programming: Java, Python, C, .NET
- Database: SQL, MySQL, Oracle
- Testing: manual, Selenium (if QA role)
- Cloud: AWS/Azure basics (bonus)
- Soft: communication, teamwork (in summary only — not as skills spam)

## Mass Recruiter 10-Second Scan Order

1. **College name + CGPA** — eligibility
2. **Skills** — keyword match?
3. **Projects** — real or filler?
4. **Internship** — any industry exposure?
5. **Format** — parse errors = skip
