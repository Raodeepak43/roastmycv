Your **campus placement resume** is not the same document you send on LinkedIn after graduation. TPO portals, company-specific formats, and batch-wide competition change the rules. One weak section — wrong CGPA format, missing project detail, photo in header — and your CV never reaches the recruiter even if you are technically qualified.

These **campus placement resume tips** cover what Training & Placement Officers (TPOs) and company HR teams actually filter for in 2026, from tier-2 engineering colleges to IIT/NIT batches targeting product companies.

## How Campus Placement Screening Works

Understanding the funnel helps you optimize each stage:

1. **TPO collection** — You upload to college portal (often strict PDF naming: `RollNumber_CompanyName.pdf`)
2. **TPO shortlist** — Coordinator removes incomplete, badly formatted, or off-policy CVs
3. **Company ATS upload** — HR bulk-imports hundreds of resumes; parsing failures auto-reject
4. **Recruiter scan** — 6–10 seconds per CV for aptitude test / interview shortlist
5. **Interview panel** — Finally, humans read details

Most students optimize for step 5 while failing step 3. Fix ATS and format first — see [ATS-friendly resume guide](/blog/ats-friendly-resume).

## TPO Resume Format Rules (Non-Negotiable)

Rules vary by college — always read your TPO's official PDF. Common requirements across Indian campuses:

| Rule | Why it matters |
|------|----------------|
| **1 page for BTech freshers** | Two pages rarely needed without work experience |
| **No photo** (many IT companies) | Infosys, TCS, Wipro often reject photos for diversity compliance |
| **Consistent file naming** | Wrong name = lost in folder of 400 CVs |
| **CGPA to 2 decimals** | 8.456 vs 8.46 — follow portal spec |
| **10th / 12th percentages** | Still requested by service companies |
| **No spelling errors in college name** | Embarrassingly common reject reason |
| **Single column layout** | Tables break company ATS |

When in doubt, ask your TPO coordinator once — then batch-mates copy you forever.

## Sections Every Placement Resume Needs

### 1. Header (Contact Block)

- Full name (as per 10th marksheet)
- Phone (Indian mobile, no +91 confusion in parsing)
- Professional email (not coolboy1999@…)
- LinkedIn URL (clean, updated)
- GitHub / portfolio (mandatory for CS/IT — optional for others)
- **City only** — full address unnecessary

### 2. Professional Summary (2–3 Lines)

Not "career objective." State degree, batch, target role, one proof point.

Example:
> B.Tech ECE 2026 (CGPA 8.2) with embedded systems project experience (Arduino, C) and 6-month IoT internship. Seeking core electronics or embedded firmware roles.

More templates: [resume summary examples](/blog/resume-summary-examples-india).

### 3. Education (Reverse Chronological)

For each degree:
- College name, degree, branch
- CGPA or percentage (both if company asks)
- Year of passing (expected: May 2026)
- 10th and 12th board marks (if space — required by many service MNCs)

**Tip:** If CGPA below 7.0, do not hide it — HR verifies. Compensate with projects and internships.

### 4. Technical Skills (Grouped)

Group logically — do not list 40 tools:

```
Languages: C, C++, Python, Java
Web: HTML, CSS, JavaScript, React
Databases: MySQL, MongoDB
Tools: Git, VS Code, Linux
```

Mirror job description keywords honestly. Keyword stuffing without interview backup fails in technical rounds.

### 5. Projects (Most Important for CS/IT)

Minimum **2 strong projects**, each with:
- Project title (link if live)
- Tech stack
- 2–3 bullets: problem, your role, outcome with numbers

**Bad:** "Library Management System using Java"

**Good:** "Built library management web app (Java, Spring Boot, MySQL) for 300+ book catalog — reduced manual issue logging time from 5 min to 30 sec per transaction"

Freshers with weak internships: projects carry your entire technical credibility.

### 6. Internships / Training

Even 1-month virtual internship counts if you describe real work. Company name, duration, 2 bullets with deliverables.

No internship? Double down on projects + hackathons + open source.

### 7. Achievements / Certifications (Optional but Valuable)

- Hackathon ranks (top 10/500 teams)
- Google Cloud / AWS / NPTEL certifications (relevant ones only)
- Competitive programming profiles (CodeChef rating if respectable)
- Leadership ONLY if tied to outcomes (GDSC Tech Lead → organized 5 workshops, 200 attendees)

Skip: "Winner of college rangoli competition" unless applying to creative roles.

## Company-Type Strategy: Service vs Product

### Service Companies (TCS, Infosys, Cognizant, Wipro, Capgemini)

- Emphasize **CGPA, 10th/12th, communication**
- Clean conservative format
- Willingness to relocate / work shifts (if true, state once)
- Certifications in Java, Python, SQL valued
- Mass apply — volume game, resume must be flawless not fancy

### Product / Startup (Flipkart, Razorpay, Swiggy, Zomato, unicorns)

- Emphasize **projects, GitHub, impact metrics**
- CGPA threshold exists but projects break ties
- System design basics for backend roles
- Show product thinking in project descriptions

### Core Engineering (Manufacturing, Auto, PSU)

- CAD tools, shop floor exposure, industrial training
- Core subject grades matter
- Location preferences relevant

Tailor one master resume lightly per track — do not maintain three completely different identities.

## Top 10 Campus Placement Resume Mistakes (2026)

1. **Sending .docx when portal asks PDF** (or vice versa)
2. **Photo on IT company application** when prohibited
3. **"Skills: MS Office, Internet, Hard Working"** — empty filler
4. **Copy-paste project descriptions** identical to three batchmates — plagiarism checks exist
5. **Broken GitHub links** — 404 repos destroy trust
6. **Listing 15 programming languages** — interview disaster waiting
7. **No page numbers** on 2-page CVs when allowed
8. **Typos in company name** on cover email ("Infoysis")
9. **Missing backlogs** — disclose if asked; lying fails background verification

## After TPO Shortlist: Interview Round Prep

Resume gets you aptitude test and HR screen — not the job. Once shortlisted:

- Every bullet on CV is fair game in technical interview
- "Basic" skills listed will be tested

## FAQ

### One page or two pages for BTech placement resume?

One page default for freshers without full-time experience. Two pages only if you have substantial internships, research papers, or multiple major projects that need space — and your TPO allows it.

### Should I include hobbies?

Only if memorable and conversation-worthy (national-level sport, significant volunteer work). Skip "reading, music, travelling" — wastes space.

### Can I use Canva resume templates?

Visual templates often break ATS. Use Google Docs, Word, or LaTeX single-column templates. Pretty ≠ parseable.

### What if I have backlogs?

Clear them before placement season if possible. If active backlog, check company eligibility rules — many MNCs require zero active backlogs. Do not lie on forms.

### Does TPO rank matter on resume?

Mention if top 5–10% (Department rank 3/120). Bottom half — omit ranking, lead with projects.

### How is campus CV different from off-campus?

Campus: standardized, batch competition, often CGPA gates. Off-campus: project proof and referrals matter more. Keep master CV, adjust emphasis.
