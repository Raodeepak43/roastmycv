## TCS NQT Resume — Section Order

```
1. CONTACT (name, phone, email, city — no full address needed)
2. EDUCATION (degree, college, CGPA, year, backlogs if any)
3. TECHNICAL SKILLS (programming languages, DB, tools — match JD)
4. PROJECTS (2–3 with stack, role, outcome)
5. INTERNSHIPS / TRAINING
6. CERTIFICATIONS (NPTEL, Coursera — only if relevant)
7. ACHIEVEMENTS (coding contests, hackathon rank)
```

**One page.** Recruiters at TCS scale do not flip pages for freshers.

## TCS NQT Format Rules

| Rule | Why |
|------|-----|
| Single column | Taleo/iCIMS parsers read top-down |
| No photo | Standard MNC policy + parser risk |
| PDF text-based | Image PDFs = zero parsed keywords |
| CGPA visible | Eligibility screening |
| Metrics in bullets | "Improved query time 40%" beats "worked on DB" |
| Role keywords | Java, Python, SQL, testing basics per JD |

## Common TCS NQT Resume Rejects

1. **"Career objective: To work in a reputed organization"** — delete
2. **Skills list without projects** — proof missing
3. **Final year project with no tech stack** — "Library Management System" alone fails
4. **Two-column Canva template** — parser scrambles text
5. **CGPA in footer** — eligibility miss
6. **Same CV for TCS Digital vs Ninja** — tailor keywords
