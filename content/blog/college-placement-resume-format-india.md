## TPO-Approved Section Order

```
1. CONTACT
   Name | Phone | Email | LinkedIn | GitHub (tech roles) | City

2. SUMMARY (2 lines max)
   Target role + strongest proof (project/internship/CGPA)

3. EDUCATION
   Degree, College, CGPA/%, Year, relevant coursework (optional)

4. TECHNICAL SKILLS
   Grouped: Languages | Frameworks | Tools | Soft skills (max 5)

5. PROJECTS (2–3)
   Name | Stack | Your role | Metric/outcome

6. INTERNSHIPS / TRAINING
   Company | Duration | 2 bullets with tasks + result

7. ACHIEVEMENTS / CERTIFICATIONS
   Hackathons, NPTEL, coding contests — with rank/scale

8. (Optional) POSITIONS OF RESPONSIBILITY
   Club lead, fest coordinator — only if space left
```

## Format Rules That Get You Rejected

| Mistake | Why TPO/ATS kills it |
|---------|----------------------|
| Two columns | Parser reads columns out of order |
| Photo / graphics | Illegal in some MNC pipelines; breaks parsers |
| "Career objective" fluff | Wastes 6-second scan window |
| Skills without proof | "Python, Java, C++" with zero projects |
| CGPA hidden | Recruiter assumes worst |
| 2+ pages (fresher) | Mass drive recruiters don't flip |

## CGPA — Where and How

Put CGPA in **Education** line — visible, not footer:

```
B.Tech Computer Science | XYZ College | CGPA: 8.4/10 | 2026
```

Rules: [CGPA on resume India](/blog/cgpa-on-resume-india-rules)

## One-Page Placement Format (Visual)

- **Margins:** 0.5–0.75 inch
- **Font:** Arial, Calibri, or Times 10–11pt
- **Headings:** Bold, ALL CAPS or Title Case — consistent
- **Bullets:** Start with action verb + metric
- **File:** PDF (text-based, not image scan)
